# 実運用前チェックリスト

ひだまり健康チェックを現場で使い始める前に、管理者が確認する項目です。
DB構造や保存仕様を変えず、画面上で安全に確認できる範囲に絞っています。

## 1. ログイン・権限

- 初期アカウントでログイン後、必ず別のパスワードへ変更できる。
- 職員アカウントで、日々の入力画面だけが迷わず開ける。
- 管理者専用画面は、職員アカウントでは開けない。
- ログイン失敗を複数回行った場合、ロック案内が表示される。

## 2. 健康チェック入力

- 記録日、利用者名、入力者を確認してから保存する運用になっている。
- 同じ記録日・利用者名を選ぶと、既存データの更新案内が表示される。
- 体温、SpO2、血圧、脈拍に明らかな入力ミスがある場合、保存前に警告またはエラーが出る。
- 体重は未測定なら空欄で保存できる。
- 保存後、同じ日付・利用者名を開き直して入力内容が表示される。

## 3. 排泄・申し送り

- 排泄チェックは、尿量と尿性状、便量と便性状の組み合わせエラーが表示される。
- 業務全体申し送りは、保存後に管理者ダッシュボードや一覧へ反映される。
- 未対応、至急、要確認の申し送りがダッシュボードで確認できる。

## 4. Supabase と SQLite

- 管理者の「セキュリティ・保守管理」から Supabase 接続診断を確認する。
- Supabase未設定の場合、SQLite側で入力・表示が継続できることを確認する。
- Supabase一時エラー時の案内が、職員にとって「入力を続けてよいか」分かる表現になっているか確認する。
- SQLiteバックアップDBの警告が出た場合、管理者へ連絡する運用を決めておく。

## 5. バックアップ・復旧

- 手動バックアップを作成し、ZIPをダウンロードできる。
- バックアップ整合性チェックで、Supabase主要データの件数を確認できる。
- 復元を試す場合は、テスト環境または復元前バックアップ作成後に行う。
- 起動時異常画面が出た場合、管理者が最新バックアップ候補を確認できる。

## 6. 写真・AI・PDF

- 写真を添付した申し送りで、圧縮後プレビューと保存サイズ案内が表示される。
- 写真保存に失敗した場合、別の写真で再試行する案内が出る。
- OpenAI APIキー未設定時でも、ルールベースの確認機能は使える。
- PDF作成に失敗した場合、エラー文を管理者へ共有し、先に画面上の文章またはExcel出力で代替する。

## 7. 朝の確認

- 管理者ダッシュボードで「今日やること」が表示される。
- 最近の注意点、排便3日なし、体重未測定、未対応申し送りが確認できる。
- 表示が空の場合も「問題なし」なのか「データなし」なのかを職員が判断できる。

## 8. 本番開始前の最終確認

- `python tools/minimal_check.py` を実行し、構文チェックがOKである。
- 実行環境に `requirements.txt` のライブラリが入っている。
- Streamlit Secrets に Supabase と OpenAI の設定が正しく入っている。
- 初日の入力担当者、バックアップ担当者、復旧判断者を決めている。
- うまく保存できない場合の連絡先と、代替記録方法を決めている。
